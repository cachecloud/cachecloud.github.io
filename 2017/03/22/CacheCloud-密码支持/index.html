CacheCloud 1.3-密码支持

发表于 2017-03-22   |   分类于 cachecloud   |     |   阅读次数

一、CacheCloud为什么之前不支持密码

  • Redis是为内网设计的。
  • “keep it simple”是Redis一直倡导的，密码会增加复杂性和成本(服务端和客户端)。
  • Redis性能很高，简单的密码很容易被暴力破解。
  • 很多公司不使用密码。

二、CacheCloud为什么现在要做

  • 现今很多Redis的公有云服务。
  • Redis出现过安全漏洞：当然这个和自身机房安全级别也有关系。
  • CacheCloud可以满足多方面的环境需求(例如接入已经存在)。
